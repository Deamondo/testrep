1. pwd = посмотреть текущий путь
2. ls -a = посмотреть список всего что лежит в текущей папке включая скрытые 
3. ls -F = выделить папки слешом /
4. history = история команд
5. grep 'blablabla' filename.txt = поиск слов в файлах
6. grep -w = поиск слова целиком
7. cat filename = посмотреть содержимое файла
8. cat > filename  = записать в файл новое содержимое, старое удалится. Завершить Ctrl+D
9. cat >> filename = дописать в файл, старое НЕ удалится. Завершить Ctrl+D
10. вся история консольных команд хранится в файле .bash_history
11. mv old_name new_name = переименовать папку
	rm -fr /path/to/folder/name/ -> удалить папку с вложеными файлами, -f - не спрашивать подтверждения для каждого файла
12. cat /var/log/dmesg|ccze -A |less -R
13. touch filename = создать файл
14. apt list code - проверка установки VSCode
15. sudo dd if=/dev/zero of=file.img bs=1M count=512 -> dd -  утилита для копирования образов и дисков в файл и из файла в любом направлении, копирование идет блоками. if - источник,  of - назначение. bs - размер блока (M = 1024 килобайт). count - количест во таких блоков
16. sudo dd if=/dev/zero of=/dev/sdb -> затрет весь диск нулями
17. ctrl + L -> очистить терминал (clear)
18. wget -> тула для закачивания файлов. wget http://www.der-hammer.info/terminal/hterm.tar.gz например
19. ls -al /dev/pts - список открытых терминалов. Можно из HTerm законнектиться к dev/pts/0 например, и получать и отправлять данные в терминал
20. sudo dmesg | grep ttyS - сообщения ядра по всем ttyS. Так можно посмотреть COM-порты
	sudo cat /proc/tty/driver/serial - тоже способ
	grep serial /proc/ioports - и это тоже
21. ln -s /usr/lib/x86_64-linux-gnu/libpng16.so.16 /usr/lib/x86_64-linux-gnu/libpng12.so.0 -> создать софт линк из источника в назначение
	ln -s ~/Qt5.12.8/Tools/QtCreator/bin/qtcreator qtt -> создать ссылку на Qt. Если создать ее в ./.local/bin/, то можно вызывать из консоли из любого места
	ls -li qtt -> проверить номер иноды этой ссылки. 
	chown -h your_user:your_group qtt -> сменить права для линка
22. which qml -> посмотреть в какой папке установлена утилита
23. cd - -> прыгнуть в предыдущую директорию
24. diff ./bin ~/Hterm/hterm -s -> сравнить  файлы построчно, можно любые
25. sudo usermod -aG с $USER -> добавить юзера в группу dialout
	cat /etc/group -> список всех групп 
	groups -> посмотреть в каких я группах
	id -> тоже самое только с номерами
	sudo usermod -d /home/new-dir lemon -> изменить домашнюю папку на новую, создастся автоматически
26. cp <archive>/hterm ~/.local/bin -> чтобы hterm запускался из любой папки, надо его скопировать в local/bin. А чтобы был виден список всех портов tty, надо добавиться в группу dialout (см. 25)
27. tar -cf arch.tar ./01_github/ -> запаковать папку в архив
	tar -xvf archive.tar -> распаковать
	tar -cjvf archive.tar файлы -> сам tar не сжимает файлы а просто создает архив. А с такими флагами еще и сожмет с помощью bzip2
28. echo $PATH - показать пути по котым происходит поиск команд
29. Монтирование флэшки:
	 ps aux | grep sdc1 -> покажет процессы, файловую систему и папку куда смонтировано
	 fuser -v -m /dev/sdc1 -> эта тула тоже покажет процессы и пути. С ее помощью можно смотреть кто использует Tcp например и проч
	 sudo mount /dev/sdc1 /media/usb -> смонтировать sdc1 в папку usb (ее надо сделать заранее). Как только она вставлена то монтируется по умолчанию сразу в /media/lemon/
30. В nano удалить строку -> Ctrl-K
31. fuser -l -> показывает все сигналы которые эта тула может послать процессу
32. chmod a-x example.txt -> сменить права доступа. a - для всех групп (могут быть еще u, g, o). -x - отнять права на выполнение. Вообще права распределены на три группы - user, group, все остальные. Отображаются в виде rwx. Например rwxr-xr--.
33. sudo cat /etc/sudoers -> в строке "Defaults	secure_path" показаны пути из которых будут запускаться проги с использованием sudo. Из обычной home/.local/bin не запустится с sudo
	sudo visudo -> тоже самое можно посмотреть  с помощью этой команды
	sudo env -> тоже можно посмотреть PATH для sudo
	разместил софтлинк на Qt в папке /usr/local/bin под названием sqt
34. df -h /usr -> можно посмотреть на каком диске физически находится папка
